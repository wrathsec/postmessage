Scenario & Vulnerability Context

An attacker hosts this page to lure a target user into initiating a connection. The attacker claims that clicking the button will synchronize or verify their secure session config details with Glia.

When the button is clicked, the real app.glia.com/account login/entry page spawns in a new tab. Because of the browser's default behavior, the first automated redirect attempt triggers a "Popup Blocked" alert. The attacker instructs the user to allow this popup manually via their browser address bar.

Because the warning notification explicitly claims that the popup is arriving from the trusted domain (app.glia.com) rather than the attacker's origin, the user will not suspect anything malicious. The moment they allow the prompt, the second delayed message delivers the payload, maintaining an active, unsevered window.opener channel that forces Reverse Tabnabbing.
